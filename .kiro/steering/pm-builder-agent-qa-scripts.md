# PM Builder Agent — Test Script Generation

## Overview

After running a QA pass (or from a BRD/PRD), the agent can generate **exportable, runnable test scripts** that engineering integrates into their CI/CD pipeline. These are traditional automation scripts — not live agent interactions.

## Supported Frameworks

| Framework | When to Use | Output File |
|-----------|------------|-------------|
| **Cypress** | Amazon frontend apps (GRC, Policy 2.0, Console) | `.cy.ts` |
| **Playwright (TypeScript)** | Cross-browser, modern apps | `.spec.ts` |
| **Playwright (Python)** | Teams preferring Python | `test_*.py` |
| **Jest + Testing Library** | Component-level unit tests | `.test.tsx` |

**Default:** Cypress (most common in Amazon frontend repos). Agent asks if unsure.

## Trigger Phrases

| Say | What Happens |
|-----|-------------|
| "Generate test scripts" | Creates scripts from the last QA run's findings |
| "Generate Cypress tests for this" | Cypress specifically |
| "Generate Playwright tests" | Playwright TypeScript |
| "Write automated tests from the BRD" | Creates tests from BRD acceptance criteria |
| "Create E2E tests for [feature]" | Full E2E test suite from workflow |
| "Export the QA scenarios as scripts" | Converts YAML scenarios to runnable code |

## Generation Sources

The agent generates scripts from THREE possible sources:

### Source 1: Last QA Run (Findings → Regression Tests)
After a QA run identifies issues, generate scripts that will CATCH those issues if they regress:

```
"Generate test scripts from the last QA run"
→ Creates one test per finding
→ Each test validates the expected behavior that was broken
→ When engineering fixes the bug, the test passes
→ If it regresses, the test fails in CI
```

### Source 2: BRD Acceptance Criteria (Spec → Tests)
From the `brd-acceptance-criteria.yaml` registry:

```
"Write automated tests from the BRD"
→ Creates one test per acceptance criterion
→ Covers happy path + failure cases
→ Maps to Epic numbers for traceability
```

### Source 3: QA Scenario Library (YAML → Code)
Converts `docs/qa-scenarios/*.yaml` to executable scripts:

```
"Export the QA scenarios as scripts"
→ Each YAML scenario becomes a test function
→ Assertions map to expect() calls
→ Cleanup steps become afterEach() hooks
```

Scripts are saved to `docs/test-scripts/[framework]/`:

## Script Quality Standards

Every generated script must:

1. **Be immediately runnable** — no missing imports, no broken selectors
2. **Use data-testid or accessible selectors** — prefer `[data-testid="..."]`, `getByRole()`, `getByText()` over fragile CSS paths
3. **Handle async properly** — `await` all interactions, explicit waits for network/render
4. **Be self-documenting** — test name describes the scenario, comments explain non-obvious steps
5. **Include setup and teardown** — `beforeEach`/`afterEach` for state management
6. **Map to findings/criteria** — each test references the finding ID or BRD criterion it validates
7. **Handle auth** — include Midway auth setup for internal Amazon apps (via plugins)

## Cypress Template (Amazon Internal Apps)

```typescript
// File: docs/test-scripts/cypress/[feature]-[scenario].cy.ts
// Generated by: pm-builder-agent
// Source: [QA Finding #X | BRD Epic-Y.Z | Scenario: "name"]
// Date: [generation date]

describe('[Feature]: [Scenario Name]', () => {
  beforeEach(() => {
    // Auth: Use Midway plugin for internal apps
    cy.midwayLogin();
    
    // Navigate to the page under test
    cy.visit('[URL]');
    
    // Wait for page to fully load
    cy.get('[data-testid="page-content"]', { timeout: 15000 }).should('be.visible');
  });

  it('[test description mapping to acceptance criteria]', () => {
    // Step 1: [action description]
    cy.get('[data-testid="element"]').click();
    
    // Step 2: [action description]
    cy.get('[data-testid="element"]').type('value');
    
    // Assertion: [what should be true]
    cy.get('[data-testid="result"]').should('contain.text', 'expected');
  });

  it('[negative case / error state]', () => {
    // Test the failure path
    cy.get('[data-testid="element"]').should('not.exist');
  });
});
```

## Playwright Template (TypeScript)

```typescript
// File: docs/test-scripts/playwright/[feature]-[scenario].spec.ts
// Generated by: pm-builder-agent
// Source: [QA Finding #X | BRD Epic-Y.Z]

import { test, expect } from '@playwright/test';

test.describe('[Feature]: [Scenario]', () => {
  test.beforeEach(async ({ page }) => {
    // Navigate and wait for load
    await page.goto('[URL]');
    await page.waitForLoadState('networkidle');
  });

  test('[description]', async ({ page }) => {
    // Step 1
    await page.getByRole('button', { name: '[button text]' }).click();
    
    // Step 2
    await page.getByLabel('[label]').fill('[value]');
    
    // Assert
    await expect(page.getByText('[expected]')).toBeVisible();
  });

  test('[negative case]', async ({ page }) => {
    await expect(page.getByTestId('[element]')).not.toBeVisible();
  });
});
```

## Finding-to-Test Mapping

When generating from QA findings, each finding produces a specific test:

| Finding Type | Test Pattern |
|-------------|-------------|
| Content contradiction (e.g., "Yes" + 0 attachments) | Assert content matches state: `expect(recommendation).not.toBe('Yes')` when attachments = 0 |
| Jargon leak | Assert no internal terms in user text: `expect(insightText).not.toContain('ConfirmationField')` |
| Missing flag propagation | Assert flag visible inline: `expect(inlineWarning).toBeVisible()` when Full Analysis has flag |
| Confidence mismatch | Assert recommendation adapts: if confidence = Low → recommendation ≠ unconditional "Yes" |
| Evidence not validated on upload | Assert validation triggers: after file upload → wait → check for validation result |
| Missing UI element (readiness status) | Assert element exists: `expect(readinessStatus).toBeVisible()` |

## BRD-to-Test Mapping

Each BRD acceptance criterion generates a test with:
- **Test name** = BRD requirement title
- **Describe block** = Epic name
- **Assertions** = directly from acceptance criteria "when/then" language
- **Comment** = links back to BRD criterion ID

Example:
```typescript
// BRD: Epic-1.1, Criterion: "Change detection within 3 seconds"
describe('Epic-1.1: Pre-fill and Variance Detection', () => {
  it('E1.1-01: Change detection completes within 3 seconds', () => {
    const startTime = performance.now();
    cy.visit('/docs/Review/407759/draft');
    cy.get('[data-testid="clara-insights"]', { timeout: 3000 }).should('be.visible');
    const elapsed = performance.now() - startTime;
    expect(elapsed).to.be.lessThan(3000);
  });
});
```

## Scenario-to-Test Mapping

YAML scenarios convert to code:

```yaml
# Input (YAML)
- name: "Invalid evidence — wrong name"
  steps:
    - action: attach_file
      params:
        name: "Random Evidence Test"
    - action: wait_for_reanalysis
  assertions:
    - surface: inline_q3
      check: contains_evidence_warning
      expected: true
```

```typescript
// Output (Cypress)
it('Invalid evidence — wrong name: inline Q3 shows warning', () => {
  // Attach file with wrong name
  cy.get('[data-testid="add-attachment"]').click();
  cy.get('[data-testid="file-name-input"]').type('Random Evidence Test');
  cy.get('[data-testid="upload-button"]').attachFile('random-evidence.pdf');
  
  // Trigger re-analysis
  cy.get('[data-testid="reanalyze-button"]').click();
  cy.wait(30000); // Wait for analysis
  
  // Assert: inline Q3 shows evidence warning
  cy.get('[data-testid="q3-clara-card"]')
    .should('contain.text', 'evidence')
    .and('contain.text', 'invalid');
});
```

## Auth Patterns for Amazon Internal Apps

### Cypress + Midway

```typescript
// cypress/support/commands.ts
Cypress.Commands.add('midwayLogin', () => {
  // Uses @amzn/cypress-midway-plugin
  cy.midway(); // or cy.authenticate() depending on plugin version
});
```

### Playwright + Midway

```typescript
// playwright.config.ts
export default defineConfig({
  use: {
    storageState: '.auth/midway-state.json', // Saved from manual login
  },
});

// global-setup.ts
async function globalSetup() {
  // Authenticate once, save state for all tests
  const browser = await chromium.launch({ headless: false });
  const page = await browser.newPage();
  await page.goto('https://gamma.grc.a2z.com');
  // Wait for Midway redirect to complete
  await page.waitForURL('**/grc.a2z.com/**');
  await page.context().storageState({ path: '.auth/midway-state.json' });
  await browser.close();
}
```

## When NOT to Generate Scripts

- Don't generate scripts for findings that require MANUAL judgment (e.g., "Is this content useful?")
- Don't generate scripts for features not yet built (mark as `test.skip` with TODO)
- Don't generate scripts for destructive tests without PM approval
- Don't include real credentials, tokens, or PII in test data

    ↓
