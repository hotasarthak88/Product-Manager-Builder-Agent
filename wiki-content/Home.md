## What It Does

| Phase | Capability |
|-------|-----------|
| 🎯 **Problem** | Define problems, challenge assumptions, validate customers, create Impact Canvases |
| 📋 **Plan** | Write PRDs, BRDs, feature specs, executive summaries |
| 🎨 **Build** | Generate interactive prototypes matching your design system |
| 🔧 **Bridge** | Scaffold production code on feature branches, hand off to engineering |
| 🧪 **QA** | Automated browser testing, BRD validation, regression detection |
| ✅ **Validate** | Multi-persona expert reviews (Developer, Product, UX, Leadership, Cross-Org) |
| 🚀 **Ship** | Track weekly progress, flag risks, assess launch readiness |
| 📚 **Learn** | Capture patterns, run retrospectives |

---

## Who Is This For?

- **Product Managers** — accelerate problem → ship lifecycle
- **Technical Program Managers** — track cross-team dependencies, run QA
- **PMT members** — generate PRDs/BRDs, validate features against specs
- **Anyone** who owns a product and wants structured, AI-assisted delivery

---

## Key Principles

1. **Autonomous discovery** — builds your context silently (team, repos, goals)
2. **Data-driven opinions** — pushes back only when evidence supports it
3. **Adaptive expertise** — calibrates to your level, operates one step ahead
4. **PM owns all decisions** — the agent informs, you decide
5. **Product-agnostic** — works for any web application with any BRD/PRD
